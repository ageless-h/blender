# ToDo List（全流程）

本目录用于把 `PLAN.md` 中的里程碑拆解成**可执行的阶段计划**与**可验收的交付清单**。

## 使用方式

- 每个里程碑（M0~M4）对应一个单独文档。
- 每个 To-Do 条目都包含：
  - 目标与范围
  - 交付物（Deliverables）
  - 交付检查（Review Checklist）
  - 测试/验收（Test & Acceptance）

建议在实现过程中：

- 每完成一个 To-Do，就在对应阶段文档里把该条目标记为完成（`[x]`）。
- 每次提交（commit/PR）都能指向某个 To-Do 的交付物与验收步骤。

## 通用 Definition of Done（适用于所有阶段）

- 文档与代码的变更范围与阶段目标一致，没有混入无关重构。
- 对用户行为有明确规则：
  - 何时执行
  - 何时不执行
  - 失败/超时如何恢复
- 不阻塞 UI：
  - 任何潜在阻塞 I/O 都有后台执行/异步读取策略。
- 安全默认值正确：
  - 风险操作默认需要确认
  - 可撤销/可回退（能集成 Undo 的必须集成）

## 通用测试分层（建议）

- **T0：静态检查**
  - 能构建（或至少能通过对应模块的编译）
  - 格式化/静态检查（如项目已有 lint 规范则遵循）
- **T1：最小功能手工测试**
  - 在 Blender UI 中可复现/可验证
- **T2：回归测试（手工）**
  - 关键已有能力不退化（复制/粘贴/历史/补全/选择等）
- **T3：性能与稳定性**
  - 长输出、长历史、快速输入不会卡顿/崩溃
- **T4：跨平台一致性（按阶段需要逐步覆盖）**
  - Windows/macOS/Linux 在输入法、剪贴板、快捷键上的一致性验证
